# AppForge v0.1.0

Release date: March 13, 2026

## Summary

`v0.1.0` is the first public AppForge release. It packages the native macOS shell, pluggable AI provider routing, portable workspace management, and the first complete built-in app recipe.

## Highlights

- Native SwiftUI macOS workspace for planning, inspecting files, viewing build logs, and launching generated apps
- Provider selection for OpenAI, Anthropic, Ollama, and LM Studio
- Keychain-backed storage for cloud credentials
- Local build pipeline based on `xcodegen` and `xcodebuild`
- Playable built-in Sudoku app generation path
- Runtime palette switching and local tooling diagnostics

## Verification

The release commit was verified with:

- `xcodegen generate`
- `xcodebuild -project AppForge.xcodeproj -scheme AppForge -showBuildSettings`
- `xcodebuild -project AppForge.xcodeproj -scheme AppForge -configuration Debug -destination 'platform=macOS,arch=arm64' build`
- `xcodebuild -project AppForge.xcodeproj -scheme AppForge -configuration Release -destination 'platform=macOS,arch=arm64' build`
- Launch verification of the built `AppForge.app`

## Known Limitations

- Arbitrary prompts are still scaffold-first; the full autonomous multi-file coding loop is not complete yet.
- Xcode coding intelligence is not integrated into AppForge.
- iPhone, iPad, and watch app generation are not yet end-to-end.

## Upgrade Notes

- This release introduces bundle version `0.1.0 (1)`.
- Existing local workspaces under `~/AppForge` remain compatible.
- Users still need Xcode and XcodeGen installed locally to build generated apps.
- Packaged app assets are local release builds and are not notarized yet.
